# Handoff - 2026-05-10

## Release writing style to preserve

- For Korean community posts, do not use overly formal release-note wording.
- Do not start with "늦은" unless the user explicitly asks for that context.
- Preferred style is casual but structured.
- Start with a short drop/update line, for example `ScriptPlayer+ v0.2.3 업데이트 올렸습니다.`
- Then one short paragraph explaining the release focus.
- Then one sentence summarizing the version scope, for example `이번 v0.2.3은 ... 전반적으로 안정화한 업데이트임`.
- Use emoji section headers.
- Use short line breaks instead of dense paragraphs.
- Use plain explanatory bullets under each section.
- End important sections with an arrow summary line, for example `👉 FIT 동작이 더 예측 가능하게 변경됨`.
- End with a `🔗 링크` section.
- Keep links formatted as:
- `GitHub: https://github.com/sioaeko/scriptplayer-plus`
- `Release: https://github.com/sioaeko/scriptplayer-plus/releases/tag/vX.X.X`
- `Feedback : https://sioaeko.github.io/scriptplayer-plus-feedback/index.html`

## English Patreon / ES style

- English posts should mirror the same structure, not become corporate changelogs.
- Preferred title format:
- `ScriptPlayer+ vX.X.X Release - Feature A / Feature B / Fixes`
- Opening can be:
- `vX.X.X drop`
- Then:
- `This version is mostly a feedback-based update.`
- Keep emoji section headers and arrow summary lines.
- Use `Links` at the end.

## Important tone notes

- Avoid stiff phrasing like "This release focuses on real-world usability fixes" unless needed.
- Avoid compressing everything into a dry bullet list.
- The user's preferred tone is closer to casual dev update notes.
- Korean wording can use `~임`, `~함`, `올렸습니다`, and short direct explanations.
- Keep section spacing roomy.
