## Supported script conventions

The most reliable multi-axis layout is separate files with matching media basename:

```text
video.mp4
video.funscript          -> L0 / Stroke
video.roll.funscript     -> R1 / Roll
video.pitch.funscript    -> R2 / Pitch
video.twist.funscript    -> R0 / Twist
video.surge.funscript    -> L1 / Surge
video.sway.funscript     -> L2 / Sway
```

Known limitation:

- A single combined multi-axis funscript file may not be parsed correctly unless it matches the app's current internal bundle shape.
- Current basic parser mostly expects normal funscript `actions`.
- If a user reports combined multi-axis not loading, ask for a sample or redacted structure before implementing support.

## Current known/pending issues for v0.2.6

### 1. Fullscreen dropdown/menu cannot select items

Reported by Noname9.

Likely cause:

- Fullscreen popup/drawer close behavior is too aggressive.
- `mouseleave` closes the drawer/menu while the user is interacting with a dropdown or native menu, especially on Linux/Wayland or multi-monitor setups.

Likely fix direction:

- Do not close immediately on `mouseleave`.
- Keep drawer/menu open while interacting with dropdown/menu content.
- Prefer outside click, ESC, or delayed close.
- Test with fullscreen and second monitor to the left.

### 2. SR6 pause jitter after about 0.5 seconds

GitHub issue:

- `https://github.com/sioaeko/scriptplayer-plus/issues/13`

Reported behavior:

- Connect SR6.
- Play single-axis or multi-axis script.
- Pause midway.
- Device appears to stop, then after about 0.5s jitters briefly at high speed.

Likely cause:

- Normal pause stops stream but then sends an extra neutral/home/keep-alive command.
- SR6 moves quickly to that command position.

Likely fix direction:

- Normal pause should stop active stream only.
- Do not send home/neutral movement on normal pause.
- Home/neutral should be reserved for disconnect/reset/explicit stop if needed.
- Prevent paused idle keep-alive from sending a movement frame immediately after pause.

Already commented on issue #13 asking for:

- Intiface or direct serial COM
- selected compatibility preset
- whether it happens every time
- whether it is worse far from center

### 3. Green script icon but script does not auto-load

User report:

- Green icon appears beside filename, meaning script detected.
- But script is not loaded automatically; user must select manually.

Likely cause:

- Detection and auto-load are separate.
- `hasScript`/`scriptAxes` can be true while `autoScriptPath` is missing or ambiguous.
- Possible cases:
  - multiple script variants
  - similar but not exact filenames
  - separate script folder
  - network/NAS paths
  - multi-axis files with no clear L0/primary path

Fix direction:

- If green icon appears, app should either auto-load a representative script or show why it cannot.
- If multiple candidates exist, show "Script detected, multiple candidates found" and offer selection.
- Align scan detection logic with load logic.

### 4. Combined multi-axis script support

User asked why combined multi-axis scripts only load L0 or lose stroke when using "multi-axis only".

Current explanation:

- App best supports split axis files by suffix.
- Single-file combined multi-axis format may not be supported depending on structure.
- "Multi-axis only" is a filter, not a playback mode.

Fix direction:

- Rename "Multi-Axis Only" to something like "Show multi-axis files only".
- Add combined multi-axis parser only after getting real samples.
